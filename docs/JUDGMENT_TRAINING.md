# 투자 판단 훈련 설계

## 목표

사용자가 과거 특정 시점의 정보만 보고 투자 가설을 작성하고, 사업 이해·근거·경쟁우위·위험·판단 변경 기준을 반복 훈련하도록 합니다. 높은 수익률을 맞히는 게임이 아니라 **재현 가능한 판단 과정**을 교육하는 기능입니다.

## 사용자 흐름

1. 과거 기업·시장 시나리오를 선택합니다.
2. 당시까지 공개된 가격, 뉴스, 재무 지표와 원문 출처를 읽습니다.
3. 다섯 가지 체크리스트와 행동·보유 기간을 제출합니다.
4. 서버가 100점 루브릭으로 사고 과정을 채점합니다.
5. 제출 후에만 해당 기간의 사후 수익률을 별도 참고 정보로 공개합니다.
6. 기록 화면에서 자신의 판단 기준이 개선되는지 비교합니다.

## 안전 장치

- 사후 수익률은 제출 전에 API 응답에도 포함하지 않습니다.
- 각 출처의 발행일은 시나리오 정보 기준일보다 늦을 수 없으며 테스트로 차단합니다.
- 사후 수익률은 점수에 사용하지 않습니다.
- AI는 점수를 변경하지 않고, 서버에서 결정된 항목별 점수를 설명만 합니다.
- AI 키는 클라이언트에 두지 않습니다. AI가 꺼졌거나 실패하면 결정론적 피드백으로 동작합니다.
- 모든 화면과 피드백은 교육용이며 실제 투자 권유가 아니라는 고지를 표시합니다.
- 시나리오에는 대표 사건 원문과 구조 검토일을 표시합니다. 프로토타입에서 이관한 가격·재무 수치는 `prototype-derived-review-required`로 표시하며, 운영 데이터로 승격하기 전에 시점·주식분할·통화·재무 분기를 공식 원문과 대조해야 합니다.

## 채점 루브릭

| 항목 | 배점 | 확인 내용 |
|---|---:|---|
| 사업 이해 | 20 | 제품·고객·수익 구조를 구체적으로 설명했는가 |
| 근거 사용 | 25 | 당시 화면의 수치와 사건을 가설에 연결했는가 |
| 경쟁우위 | 15 | 강점과 지속 가능성 또는 약화 조건을 설명했는가 |
| 위험 인식 | 20 | 기업·시장 위험과 손실 경로를 설명했는가 |
| 판단 변경 기준 | 20 | 기간, 수치, 가설 훼손 조건을 사전에 정했는가 |

시나리오별 핵심 개념과 수치·조건 인용을 사용하되, 충분한 설명 길이와 인과 표현이 없는 키워드 나열에는 항목별 점수 상한을 적용합니다. 운영 전에는 사람 채점 표본과 비교해 오탐과 집단별 편향을 측정해야 합니다.

## API

- `GET /api/judgment/scenarios`
- `GET /api/judgment/scenarios/:id`
- `POST /api/judgment/attempts`
- `GET /api/judgment/attempts/:id`
- `GET /api/judgment/history`

모든 엔드포인트는 JWT 인증을 요구합니다. 제출 API는 사용자/IP 기준 인메모리 속도 제한을 적용합니다.

## 데이터베이스

신규 설치는 `database/schema.sql`을 사용합니다. 기존 DB는 아래 마이그레이션을 적용합니다.

```bash
mysql -u root -p gdgoc_sch < database/migrations/002_judgment_training.sql
```

## AI 피드백 설정

기본값은 비활성화입니다. 서버 환경 변수로만 켭니다.

```dotenv
AI_FEEDBACK_ENABLED=true
LLM_PROVIDER=gemini
GEMINI_API_KEY=...
```

`LLM_PROVIDER=groq`도 지원합니다. 키가 없거나 호출이 실패하면 점수와 저장은 정상 진행되고 기본 피드백을 반환합니다.
